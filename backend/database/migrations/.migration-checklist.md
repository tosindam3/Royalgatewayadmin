# Migration Checklist

Before deploying migrations to production:

## Pre-Deployment
- [ ] Test migration on local database with production-like data
- [ ] Test rollback functionality
- [ ] Check for data loss scenarios
- [ ] Verify indexes won't lock tables for too long
- [ ] Ensure backward compatibility with current code
- [ ] Document any manual steps required

## Safe Migration Patterns
✅ Adding nullable columns
✅ Adding new tables
✅ Adding indexes (use `ALGORITHM=INPLACE` for MySQL)
✅ Creating new relationships

⚠️ Requires Caution
- Renaming columns (use multi-step deployment)
- Changing column types
- Adding non-nullable columns (add as nullable first, then update)
- Dropping columns (deprecate first, drop later)

## Multi-Step Deployment Strategy
For breaking changes:
1. Deploy code that works with both old and new schema
2. Run migration
3. Deploy code that uses new schema only
4. Clean up deprecated code

## Production Deployment
```bash
# Always backup first
php artisan backup:run

# Run migrations
php artisan migrate --force

# If something goes wrong
php artisan migrate:rollback --step=1
```
